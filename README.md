# Quilum

Quilum - это приложение для автоматического планирования дел и поддержания
концентрации путем блокировки отвлекающих приложений.

_Приложение находится в активной разработке, но совсем скоро будет доступно для установки. Следите за изменениями в соответствующих ветках репозитория._

## Мотивация

На рынке существует довольно много календарей или приложений для ведения задач.
Однако не все из них позволяют **размещать задачи в календаре** (встраивать их в
расписание). Кроме того, ручное планирование дел занимает неоправданно большое
количество времени, а решения, которые позволяют автоматизировать этот процесс,
зачастую очень дорогие.

Помимо этого, стресс и усталость могут привести к прокрастинации и проблемам с
концентрацией. Конечно, во время работы можно отложить телефон, но вам не
удастся отложить компьютер, за которым вы работаете, хотя и на нем может быть
множество совершенно не связанных с вашими целями приложений, которые будут
бороться за каждую минуту вашего внимания. Их неосознанное использование будет
отнимать бесценное время и приводить к еще большему истощению.
_Кроссплатформенных решений_, способных помочь решить эту проблему, а тем более
_синхронизирующих блокировку с графиком работы_ пользователя, **найдено не
было**.

_Поэтому и создается Quilum._

Автоматический планировщик, **работающий локально и не требующий никакой
инфраструктуры** (следовательно, **и затрат на нее**), позволит вам не тратить время и
энергию на составление расписания, а благодаря **автоматической блокировке
приложений на время выполнения задач** у вас _не будет технической возможности
отвлечься_, что позволит вам выполнять задачи быстрее и с меньшими затратами
энергии.

## Составление расписания и планирование задач

Приложение позволяет пользователю вести учет событий (как в календаре) и задач.

Для событий можно указывать:
- название,
- описание,
- дату и время начала,
- дату и время окончания.

Для задач можно указывать:
- название,
- описание,
- срок выполнения,
- приоритет (низкий, средний, высокий),
- примерное время выполнения.

Чтобы воспользоваться планировщиком, укажите окна в расписании, которые вы
готовы уделить выполнению задач. Эти окна называются _слотами_.

Для создания слота достаточно указать:
- время начала,
- время окончания.

Когда вы закончите, планировщик проанализирует доступные слоты и
незапланированные задачи. Он составит план, максимизирующий количество вовремя
выполненных задач, при этом приоритет отдается наиболее важным из них. Задачи,
которые не удалось вписать в расписание без нарушения сроков, не попадают в
план.

## Блокировка приложений

Одна из ключевых возможностей Quilum - когда вы выполняете какую-то задачу, не
давать вам запускать приложения, которые вас отвлекают.

Для каждой ОС спроектирована своя реализация механизма блокировки, состоящая из дополняющих друг друга слоев:
- Windows
  - Слой 1: `SetWinEventHook` - реактивное обнаружение окон
    - Подписка на события о создании окон отвлекающих приложений, немедленное
      завершение их работы при попытке запуска.
  - Слой 2: Опрашивание процессов
    - С определенным интервалом Quilum получает список работающих процессов.
      Если он находит отвлекающее приложение, оно получает сигнал завершить
      работу.
- Linux
  - Слой 1: Polkit-помощник для bind-mount
    - Используется специальный демон, который монтирует `/dev/null` по путям,
      где находятся отвлекающие приложения и таким образом "прячет" их, делая
      запуск невозможным.
  - Слой 2: Опрашивание процессов
    - По аналогии с Windows.
- macOS (в данный момент используется единственный слой)
  - Слой 1: NSWorkspace Notifications
    - Quilum подписывается на уведомления о запуске новых приложений. Если
      пользователь пытается запустить отвлекающее приложение, оно тут же
      завершает работу.
    
Подробнее о методах реализации блокировки написано в [заметках о блокировке](app-blocking-mechanisms.md).
